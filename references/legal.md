# Legal — 사우디 법적 리스크 및 준수 사항

## 주의

이 문서는 법률 자문이 아니다. 프로젝트 운영 담당자의 판단 자료일 뿐이며, 실제 배포 전 반드시 법무 검토를 받는다. 정보는 시장조사 보고서(2026년 4월 기준) 기반이므로 최신 변경 여부를 수시 확인한다.

## 적용 가능 법령 3종

### 1. PDPL (Personal Data Protection Law)
사우디 개인정보 보호법. 2023-09-14 시행, 추가 준수 유예기간 있음.

### 2. Anti-Cyber Crime Law
사우디 사이버범죄 방지법. 무단 접근·방해·데이터 탈취를 형사처벌 대상으로 규정.

### 3. Copyright Law (2026년 신 저작권법)
- 왕령 M/169 **2026-02-13 공포**
- **시행일: 공포 후 180일 경과** → 대략 2026-08-12 전후
- 기존 2003년 법 대체, 보호·집행 체계 강화

## 프로젝트에 적용되는 규칙

### PDPL 관련

**현재 수집 대상은 개인정보가 아니다.** 의약품 가격·등록번호·제조사는 법인 정보다. 그러나 다음 순간 PDPL이 강하게 적용된다:

- ❌ 사용자 처방전/진단서 업로드 기능 추가
- ❌ 의료진 연락처/이메일 수집
- ❌ 환자 후기/리뷰 데이터 크롤링
- ❌ 약국 직원 명단 수집
- ❌ SNS 게시글에서 개인 식별 정보 추출

**원칙**: 사용자 입력은 **제품 메타데이터만** 받는다. 품목/성분/함량/제형 4개 필드로 고정. 처방/인증 정보 업로드 금지.

### Anti-Cyber Crime Law 관련

**절대 하지 말 것**:

- ❌ 로그인 우회 (SQL injection, 세션 쿠키 조작, 자격증명 스터핑)
- ❌ CAPTCHA 무단 돌파 (2captcha는 공식 서비스 경로로만 사용)
- ❌ 사이트의 Rate Limit을 고의로 우회 (프록시 풀 강제 순환, 헤더 조작으로 재식별 회피)
- ❌ robots.txt `Disallow` 경로 강제 수집
- ❌ 비공개 관리자 페이지/API 엔드포인트 탐색
- ❌ 취약점 스캐닝으로 보일 수 있는 요청 패턴 (diff fuzzing, path brute force)

**원칙**: "공개된 페이지, 공식 API, 정당한 계정" 3가지만 접근한다.

### Copyright Law 관련

**2026년 신법 시행으로 크롤링 재배포 리스크가 올라간다.** 특히:

- 수집 데이터의 **외부 공개·판매 금지** (캡스톤 내부 용도만)
- 원문 HTML 대량 복제·스크린샷 저장 금지
- 카탈로그 재구성·재배포 금지 (예: 사우디 의약품 가격 DB를 외부에 판매)
- 이미지·로고 수집 금지 (상품 사진 다운로드 자제)

**안전한 저장 원칙**: 필요 최소 필드 + 출처 URL + 타임스탬프. 원문 HTML 덤프는 디버깅용으로 7일만 보관 후 삭제.

## robots.txt 해석 원칙

### 표준

**RFC 9309 (Robots Exclusion Protocol)** — 2022년 IETF 표준.

### 핵심 오해 금지

> **"robots.txt를 준수한다"는 "합법이다"와 같은 말이 아니다.**

- `Allow: /` 여도 이용약관에서 자동화를 금지할 수 있다.
- `Disallow: /private/` 여도 이 문구만으로 법적 금지가 되는 것은 아니다 (무단 접근 판단은 법원 몫).
- robots.txt는 "요청(requested) 규칙"이지 "허가(authorization) 규칙"이 아니다.

### 실무 구현 규칙

1. **Disallow 경로는 무조건 건드리지 않는다.** 법적 논란을 원천 차단.
2. **User-agent `*` 규칙을 따른다.** 우리는 특정 봇 이름을 주장하지 않는다.
3. **`Crawl-delay`가 있으면 토큰 버킷 rate를 그 값에 맞춰 낮춘다.**
4. **robots.txt 자체가 없거나 200 OK가 아니면 보수적으로 해석** (사이트 측 의사 불명확 → 낮은 rate로 진행).
5. **robots.txt 변경을 모니터링.** 24시간 TTL 캐싱, 만료 시 재조회.

## 이용약관 (Terms of Service) 체크

소스 추가 전 반드시 확인:

| 체크 항목 | 방법 |
|---|---|
| 자동화 크롤링 허용 여부 | ToS 페이지에서 "automated", "crawler", "scraping", "robots" 검색 |
| API 사용 조건 | Developer Portal 약관 |
| 데이터 재배포 허용 여부 | ToS + Open Data 라이선스 |
| 광고/요금표 재구성 금지 조항 | 소매 사이트 대부분 존재 |
| 봇 차단 회피 금지 조항 | 보안 관련 조항 |

### 소스별 체크 결과 (시장조사 기준, 재확인 필요)

| 소스 | 자동화 | API 제공 | 비고 |
|---|---|---|---|
| SFDA | ✅ (Open Data API) | ✅ | OAuth 필요, 구독 승인 후 |
| NUPCO | ⚠️ 명시 없음 | ❌ | 공개 텐더만, 로그인 영역 금지 |
| Etimad | ⚠️ 약관 확인 필요 | ✅ (유료) | 약관에 키 관리 의무 명시 |
| Nahdi | ⚠️ 명시 없음 | ❌ | 공개 상품 페이지만, 앱 API 건드리지 말 것 |
| Al-Dawaa | ⚠️ 명시 없음 | ❌ | 위와 동일 |
| Whites | ⚠️ 명시 없음 | ❌ | 위와 동일 |
| Tamer / Ultra | ⚠️ 명시 없음 | ❌ | 회사 홈페이지만 |
| Noon / Amazon.sa | ❌ **명시적 금지 가능성** | (제한적 Seller API) | 매우 보수적으로 |

**"명시 없음"**은 "허용"이 아니다. 법무 검토 필수.

## 법적 리스크 체크리스트

신규 소스 추가 시 아래 6개를 전부 통과해야 한다.

- [ ] 공개 포털/API 범위만 접근한다 (로그인 필요 내부 시스템 금지)
- [ ] 사우디 Anti-Cyber Crime Law 저촉 행위 없음 확인
- [ ] 해당 사이트 이용약관에서 크롤링 금지 조항 확인
- [ ] API 역산은 **공개 데이터 범위 내**에서만 (인증 우회 금지)
- [ ] 수집 데이터는 캡스톤 **내부 용도만** (외부 공개·판매 금지)
- [ ] PM·법무 검토 완료 및 면책 조항 명시

## 키 관리·보안 원칙

### Etimad Developer Portal 약관 사례

Etimad API 포털 약관에는 계정·키 관리 의무가 명시되어 있다(비밀번호·키 공유 금지 등). 이는 사우디 정부 포털 대부분에 적용되는 공통 원칙.

### 실무 규칙

1. **모든 API 키는 GitHub Secrets에만 저장.** 환경변수로 주입.
2. **로컬 `.env` 금지.** 개발자 머신 탈취 시 즉시 유출.
3. **키를 로그에 찍지 않는다.** `print(api_key)` 실수 금지. `loguru` 기본 마스킹 설정 활용.
4. **UI에 키 노출 금지.** 에러 메시지에서 URL 쿼리스트링으로 키가 새어나가지 않도록 필터링.
5. **권한 최소화.** SFDA/Etimad 계정은 팀 공용 법인 계정으로, 개인 계정 사용 금지.
6. **키 순환(Rotation).** 3~6개월마다 재발급 권장. 유출 의심 시 즉시.
7. **예산 모니터링.** 하드 리밋 설정. 초과 시 자동 중단.

## 데이터 보관 및 파기

### 원칙

- **최소 보관**: 필요 필드만 저장
- **출처 추적**: 모든 행에 `source_url`, `crawled_at` 필수
- **원문 덤프 임시**: `raw_fields` JSONB는 디버깅용, 7일 후 자동 파기(또는 익명화)

### 자동 파기 예시 (Supabase cron function)

```sql
-- 7일 지난 raw_fields 컬럼만 null 처리 (행은 유지)
update saudi_products
set raw_fields = null
where crawled_at < now() - interval '7 days'
  and raw_fields is not null;
```

### 캡스톤 종료 시

프로젝트 종료 시점에 수집 데이터 전체의 처분 방침을 PM이 결정한다:
- 학술 발표용 익명화 → 유지
- 내부 아카이브 → 암호화 저장
- 완전 파기 → `drop table`

**외부 공유·판매 절대 금지.** 이건 캡스톤 헌법이다.

## 비상 상황 대응

### 저작권 침해 경고 수신 시

1. **즉시 해당 소스 크롤링 중단.** GitHub Actions 워크플로우 disable.
2. **PM 즉시 보고.**
3. **법무팀 검토 전 재개 금지.**
4. **수집된 데이터 격리.** 별도 테이블로 이동 또는 삭제.

### 사이트 소유자 cease & desist 수신 시

1. **수신 즉시 중단.** IP 차단 여부와 무관.
2. **해당 도메인 영구 블랙리스트.** 코드 수정으로 재수집 원천 차단.
3. **PM·법무 보고.**
4. **이미 수집된 데이터 처분 결정.**

### 사이트가 자체적으로 차단했을 때

1. **재시도 금지.** 차단은 사이트 측 의사 표시.
2. **우회 시도 금지.** 프록시 교체, UA 변경으로 돌파하려 하지 말 것.
3. **1주일 대기.** 일시적 차단일 수 있음.
4. **여전히 차단이면 소스 폐기.** 다른 소스로 대체.
